
## 1. Сущность: Задачи (таблица `Задачи`)

Используется в методах:
- `GET /api/v1/tasks` – список задач.
- `POST /api/v1/tasks/{id}/complete` – отметка задачи выполненной.
- `POST /api/v1/onboarding/approve` – утверждение задачи.

| Поле в БД | Поле в DTO (ответе) | Преобразование |
|-----------|---------------------|----------------|
| `ID` | `id` | прямое копирование |
| `Заголовок` | `title` | прямое копирование |
| `Описание` | `description` | прямое копирование (если нужно) |
| `Статус` | `status` | перевод кодов: <br> `'новая'` → `'pending'`, <br> `'выполнена'` → `'completed'`, <br> `'ожидает утверждения'` → `'pending_approval'`, <br> `'просрочена'` → `'overdue'` |
| `Срок` | `deadline` | форматирование в ISO-строку (YYYY-MM-DD) |
| `ID_Пользователя` | `assignee_name` | `JOIN` с таблицей `Пользователи` по `ID_Пользователя`, взять поле `ФИО` |
| `ID_Шаблона` | `flow_name` | (опционально) `JOIN` с `Шаблоны чекистов` для получения названия шаблона |

---

## 2. Сущность: Заявки на оборудование (таблица `Заявки_на_оборудование`)

Используется в методе:
- `POST /api/v1/equipment-requests` – создание заявки.
- `GET /api/v1/hr/statistics` – статистика (агрегация).

| Поле в БД | Поле в DTO (ответе) | Преобразование |
|-----------|---------------------|----------------|
| `ID` | `request_id` | прямое копирование |
| `Тип_оборудования` | `equipment_type` | прямое копирование |
| `Комментарий` | `comment` | прямое копирование |
| `Статус` | `status` | перевод кодов: <br> `'новая'` → `'new'`, <br> `'одобрена'` → `'approved'`, <br> `'отклонена'` → `'rejected'` |
| `Дата_создания` | `created_at` | форматирование в ISO-строку с временем |
| `ID_Пользователя` | `user_name` | `JOIN` с `Пользователи` по `ID_Пользователя`, взять `ФИО` |

---

## 3. Сущность: Пользователи (таблица `Пользователи`)

Используется в методах:
- `GET /api/v1/mentor/mentees` – список подопечных.
- `GET /api/v1/hr/statistics` – статистика.
- Маппинг для `assignee_name` и `user_name` в других сущностях.

| Поле в БД | Поле в DTO | Преобразование |
|-----------|------------|----------------|
| `ID` | `id` | прямое копирование |
| `ФИО` | `name` | прямое копирование |
| `email` | `email` | прямое копирование |
| `Отдел` | `department` | прямое копирование |
| `ID_Роли` | `role_name` | `JOIN` с таблицей `Роли` по `ID_Роли`, взять `Название_роли` |
| `ID_Наставника` | `mentor_name` | `JOIN` с той же таблицей `Пользователи` (самосвязь) для получения ФИО наставника |

---

## 4. Сущность: Комментарии (таблица `Комментарии`)

Используется (например) для отображения комментариев к задаче (если этот метод будет добавлен позже).

| Поле в БД | Поле в DTO | Преобразование |
|-----------|------------|----------------|
| `ID` | `comment_id` | прямое копирование |
| `Текст` | `text` | прямое копирование |
| `Дата_создания` | `created_at` | форматирование даты-времени |
| `ID_Пользователя` | `author_name` | `JOIN` с `Пользователи` для получения `ФИО` |
| `ID_Задачи` | `task_id` | прямое копирование |

---

## 5. Сущность: Утверждение задач (таблица `Утверждение_задач`)

Используется для метода `POST /api/v1/onboarding/approve`.

| Поле в БД | Поле в DTO | Преобразование |
|-----------|------------|----------------|
| `ID` | `approval_id` | прямое копирование (если возвращается) |
| `Дата_утверждения` | `approved_at` | форматирование даты-времени |
| `ID_Пользователя` | `approver_name` | `JOIN` с `Пользователи` для получения ФИО утверждающего |
| `ID_Задачи` | `task_id` | прямое копирование |
