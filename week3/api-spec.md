# Спецификация API для портала онбординга

Здесь описаны основные методы, которые нужны фронтенду для работы с серверной частью. Все запросы, кроме логина, требуют токен авторизации, который передаётся в заголовке `Authorization: Bearer <token>`. Ответы приходят в формате JSON.

---

## 1. Получить список задач для текущего пользователя

**Метод:** `GET /api/v1/tasks`

**Что делает:** Возвращает все задачи, назначенные на текущего сотрудника (новичка). Используется для заполнения дашборда.

**Заголовки:**  
`Authorization: Bearer <token>`

**Параметры запроса:** не требуются.

**Тело запроса:** отсутствует.

**Тело ответа при успехе (код 200):**
```json
{
  "tasks": [
    {
      "id": 1,
      "title": "Ознакомиться с политикой безопасности",
      "deadline": "2026-07-15",
      "status": "pending"
    },
    {
      "id": 2,
      "title": "Настроить рабочую почту",
      "deadline": "2026-07-16",
      "status": "completed"
    }
  ]
}
