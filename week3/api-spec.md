# Спецификация API для портала онбординга

Здесь описаны основные методы, которые нужны фронтенду для работы с серверной частью. Все запросы, кроме логина, требуют токен авторизации, который передаётся в заголовке `Authorization: Bearer <token>`. Ответы приходят в формате JSON.

---

## 1. Получить список задач для текущего пользователя

**Метод:** `GET /api/v1/tasks`

**Что делает:** Возвращает все задачи, назначенные на текущего сотрудника (новичка). Используется для заполнения дашборда.

**Заголовки:**  
`Authorization: Bearer <token>`

**Параметры запроса:** не требуются.

**Тело запроса:** отсутствует.

**Тело ответа при успехе (код 200):**
```json
{
  "tasks": [
    {
      "id": 1,
      "title": "Ознакомиться с политикой безопасности",
      "deadline": "2026-07-15",
      "status": "pending"
    },
    {
      "id": 2,
      "title": "Настроить рабочую почту",
      "deadline": "2026-07-16",
      "status": "completed"
    }
  ]
}
```

**Возможные ошибки:**
- `401` – токен не передан или недействительный.
- `403` – у пользователя нет прав на просмотр задач (например, он не новичок).
- `500` – ошибка на сервере.

---

## 2. Отметить задачу как выполненную

**Метод:** `POST /api/v1/tasks/{id}/complete`

**Что делает:** Изменяет статус задачи на «выполнена». Обычно вызывается, когда новичок нажимает кнопку «Выполнено» на своей задаче.

**Заголовки:**  
`Authorization: Bearer <token>`

**Параметры пути:**  
`id` – идентификатор задачи (целое число, обязательное).

**Тело запроса (необязательно):**
```json
{
  "comment": "Задача выполнена, все документы подписаны"
}
```
Поле `comment` – строка, максимум 500 символов. Может отсутствовать.

**Тело ответа при успехе (код 200):**
```json
{
  "status": "completed",
  "updated_at": "2026-07-16T12:30:00Z"
}
```

**Ошибки:**
- `400` – задача уже была выполнена или невозможно выполнить действие.
- `403` – пользователь не является исполнителем этой задачи.
- `404` – задача с указанным `id` не найдена.
- `500` – внутренняя ошибка сервера.

---

## 3. Создать заявку на оборудование

**Метод:** `POST /api/v1/equipment-requests`

**Что делает:** Позволяет новичку подать заявку на получение техники (ноутбук, монитор и т.д.). Заявка сохраняется в системе со статусом «новая».

**Заголовки:**  
`Authorization: Bearer <token>`

**Тело запроса (обязательные поля):**
```json
{
  "equipment_type": "ноутбук",
  "comment": "Желательно с 16 ГБ ОЗУ и SSD"
}
```
- `equipment_type` – строка, обязательное. Допустимые значения: `ноутбук`, `монитор`, `клавиатура`, `другое`.
- `comment` – строка, необязательное, максимум 500 символов.

**Тело ответа при успехе (код 201):**
```json
{
  "request_id": 123,
  "status": "created"
}
```

**Ошибки:**
- `400` – не указан тип оборудования.
- `403` – у пользователя нет прав на создание заявки.
- `500` – ошибка сервера.

## 4. Получить список подопечных для наставника

**Метод:** `GET /api/v1/mentor/mentees`

**Что делает:** Возвращает список новичков, закреплённых за текущим наставником, с указанием процента выполнения их чек-листов. Удобно для контроля прогресса.

**Заголовки:**  
`Authorization: Bearer <token>`

**Параметры запроса:** не требуются.

**Тело запроса:** отсутствует.

**Тело ответа при успехе (код 200):**
```json
{
  "mentees": [
    {
      "id": 10,
      "name": "Анна Смирнова",
      "progress": 75
    },
    {
      "id": 11,
      "name": "Пётр Иванов",
      "progress": 40
    }
  ]
}
```

**Ошибки:**
- `401` – неавторизован.
- `403` – текущий пользователь не является наставником.
- `500` – ошибка сервера.

## 5. Утвердить завершение ключевого этапа (руководитель)

**Метод:** `POST /api/v1/onboarding/approve`

**Что делает:** Руководитель подтверждает, что важный этап онбординга (например, подписание договора) пройден успешно. После утверждения задача закрывается окончательно.

**Заголовки:**  
`Authorization: Bearer <token>`

**Тело запроса:**
```json
{
  "task_id": 5,
  "comment": "Договор подписан, замечаний нет"
}
```
- `task_id` – целое число, обязательно.
- `comment` – строка, необязательно.

**Тело ответа при успехе (код 200):**
```json
{
  "status": "approved"
}
```

**Ошибки:**
- `400` – не передан `task_id` или задача уже утверждена.
- `403` – у пользователя нет прав на утверждение (он не руководитель).
- `404` – задача с указанным `id` не найдена.
- `500` – внутренняя ошибка.

---

## 6. Получить сводную статистику по онбордингу (для HR)

**Метод:** `GET /api/v1/hr/statistics`

**Что делает:** HR-менеджер может получить обобщённые данные: сколько новичков в процессе, среднее время выполнения задач, сколько заявок на оборудование подано за период.

**Заголовки:**  
`Authorization: Bearer <token>`

**Параметры запроса (опционально):**  
`?period=week` – можно фильтровать по неделе, месяцу и т.д. Если не указан, возвращаются данные за текущий месяц.

**Тело запроса:** отсутствует.

**Тело ответа при успехе (код 200):**
```json
{
  "newbies_count": 8,
  "avg_completion_days": 4.2,
  "total_equipment_requests": 15
}
```

**Ошибки:**
- `401` – неавторизован.
- `403` – пользователь не является HR-менеджером.
- `500` – ошибка сервера.
