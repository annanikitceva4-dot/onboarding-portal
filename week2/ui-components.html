UI-компоненты системы онбординга

Компонент Где используется Назначение Состояния и поведение Ограничения
Кнопка «Выполнить» Страница чек-листа, напротив каждой задачи Отметить задачу как выполненную Активная (синяя) — по умолчанию.
Неактивная (серая) — если задача уже выполнена или просрочена.
При клике — спиннер и текст «Выполняется…».
После успеха — серая с галочкой. 		Нет ограничений
Прогресс-бар Дашборд, шапка чек-листа Показывает процент выполненных задач Заполняется от оранжевого к зелёному.
Рядом подпись «XX%».
Обновляется автоматически при изменении статуса задачи. 		Только целые числа от 0 до 100
Выпадающий список Форма запроса оборудования Выбор типа оборудования Закрыт — плейсхолдер «Выберите тип».
Открыт — варианты: Ноутбук, Монитор, Клавиатура, Мышь, Другое.
При выборе — рамка зелёная.
При ошибке (пустое поле) — рамка красная. 		Обязательное поле, по умолчанию пусто
Текстовое поле «Комментарий» Форма запроса оборудования Для дополнительной информации (модель, срочность и т.п.) Обычное — белый фон, серая рамка.
В фокусе — синяя рамка.
При превышении 500 символов — красная рамка и счётчик. 		Необязательное, максимум 500 символов
Модальное окно После отправки заявки или завершения всех задач Подтверждение успешного действия Открывается с затемнением фона, по центру.
Заголовок «Готово!», иконка-галочка, пояснительный текст и кнопка «ОК».
Закрывается по клику на «ОК» или на затемнённую область. 		Закрытие по Esc — опционально
Карточка задачи Страница чек-листа (список задач) Отображает название, дедлайн, исполнителя, статус Обычная — белый фон, тень.
Просроченная — красная полоса слева, иконка «!».
Выполненная — зелёная полоса, галочка, текст зачёркнут.
При наведении — подсветка, появляется кнопка «Выполнить» (если активна).
Клик — переход к деталям. 		Название ≤ 100 символов, дедлайн обязателен
