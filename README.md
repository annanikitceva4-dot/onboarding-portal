# onboarding-portal

**Цель проекта:** автоматизировать процесс онбординга.

## Содержание репозитория
- `week1/` – артефакты первой недели (требования, сценарии, BPMN)
  - [requirements.md](week1/requirements.md) – ФТ и НФТ
  - [user-stories.md](week1/user-stories.md)
  - [use-cases.md](week1/use-cases.md)
  - [bpmn-diagram.png](week1/bpmn-diagram.png) – диаграмма процесса To-Be
  - [questions.md](week1/questions.md) – вопросы заказчику

## Как использовать
Данные артефакты являются основой для проектирования системы на последующих неделях.

## Неделя 2: Проектирование интерфейса и архитектуры

В этой папке находятся артефакты, демонстрирующие, как будет выглядеть система и как её компоненты взаимодействуют друг с другом.

- **[Эскизы (Wireframes)](week2/)** — три ключевых экрана:
    - `dashboard.png` — дашборд новичка.
    - `check-list.png` — страница чек-листа.
    - `equipment.png` — форма запроса оборудования.
- **[Описание UI-компонентов](week2/ui-components.md)** — таблица с описанием всех элементов интерфейса, их состояний и ограничений.
- **[UML Sequence Diagram](week2/UML_sequence_diagram.drawio.png)** — диаграмма, показывающая взаимодействие фронтенда, бэкенда и базы данных при отметке задачи.
