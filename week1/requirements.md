# Требования к системе «Портал адаптации и онбординга»

## Функциональные требования (ФТ)

1. Новичок должен иметь возможность просматривать список задач, используя систему.
2. Система должна позволять новичку отмечать задачу как выполненную.
3. В системе должен отображаться прогресс выполнения чек-листа.
4. Новичок может запрашивать оборудование через форму в системе.
5. Через систему наставник может просматривать список своих подопечных.
6. Наставник через систему может оставлять комментарии к задачам новичка.
7. Через систему руководитель может видеть прогресс новичка по сделанным/просроченным задачам.
8. Система должна позволять руководителю подтверждать выполнение ключевых этапов.
9. HR-менеджер через систему может создавать готовые чек-листы (используя шаблоны).
10. 10.HR-менеджер через систему может смотреть сводную статистику (сколько новичков в онбрдинге, средняя скорость, заявки на оборудование)

## Нефункциональные требования (НФТ)

1. Безопасность: задачи видят только новичок и его наставник. используется шифрование (HTTPS).
2. Производительность: дашборд новичка загружается не более 2 секунд.
3. Удобство: важные действия должны быть доступны в 1-2 клика.
4. Надёжность: система доступна в работе 99% времени.
5. Масштабируемость: система должна выдерживать много (до 300) пользователей одновременно.
