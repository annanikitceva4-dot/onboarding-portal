# Use Case: Прохождение чек-листа первого дня

**Участники:** Новичок, Наставник  
**Условие:** Новичок залогинился в системе, чек-лист назначен.

## Основной сценарий

1. Новичок открывает главную страницу.  
2. Видит список задач (например «Прочитать регламент», «настроить рабочее место»).  
3. Выполняет задачу и нажимает «Выполнено».  
4. Статус задачи меняется на «Выполнена», прогресс-бар обновляется.  
5. Если задача требует подписи наставника (например «подписать договор»), наставнику приходит уведомление.  
6. Наставник открывает задачу, пишет комментарий «Подписано» и ставит статус «Подтверждено».  
7. Когда все задачи выполнены, чек-лист закрывается, и руководитель получает уведомление.

## Другой вариант (если задача не выполнена)

- Если новичок не может выполнить задачу (например нет доступа), он нажимает «Задать вопрос» и пишет комментарий, и наставник получает уведомление.

## Ошибки

- При сбое интернета во время отметки задачи появляется сообщение: «Ошибка, попробуйте позже».

# Use Case: Запрос оборудования (кратко)

**Участник:** Новичок  
**Условие:** новичок авторизован в системе.

## Основной сценарий

1. Новичок переходит в раздел «Запрос оборудования».  
2. Выбирает тип оборудования из списка (например ноутбук).   
3. Нажимает кнопку «Отправить».  
4. Система создаёт заявку со статусом «Новая» и показывает сообщение об успешной отправке заявки.

## Другой вариант (если поле не заполнено)

- Если новичок не выбрал тип оборудования и нажал «Отправить», система подсвечивает поле красным и выводит сообщение: «Выберите тип оборудования».

## Ошибки

- При сбое сети во время отправки заявки появляется сообщение: «Ошибка соединения, попробуйте позже».  
- Если сервер не отвечает более 5 секунд, показывается сообщение: «Сервер недоступен, повторите попытку».
